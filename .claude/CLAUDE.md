### 月面の〈黒筐〉分岐（主人公の変容ルート）
プレイヤーが月面探査中に〈黒筐（こっきょう）〉へ「触れる/触れない」で主人公の運命が二分岐:
- **黒筐ビジュアル**: 六角柱、背丈の三倍ほど。六面すべて黒い光沢を持つ鏡面。鏡面の奥には何層もの偏光膜が無限に深く透けて見え、折り目はすべて直角に畳まれている。粒子が近づけない（鏡面に届く寸前で軌道が逸れる）ため塵を一粒も乗せない。側面と底面の交差は指が切れるほど鋭利な直角
- **触れたルート（超人類化）**: 黒筐接触で超常能力が覚醒。手をかざすだけで重量物を動かす等、フォース的なテレキネシス系の力を段階的に習得していく（小説では「退ける専用×代償累進」として再構築済）
- **触れなかったルート（エイリアン化）**: 月面探査時に接触した地底エイリアンの血液が宇宙服に付着 → そこから感染。最終的に主人公自身がエイリアンへと変貌し、コロニー内の全てを捕食していくバッドエンド分岐
- ゲーム実装メモ: 現行ゲームには2001風の黒い直方体モノリスが既出（`update.js` の移動限界外、「モノリス接触でビームゲージ満タン」処理あり）。**今後の改修で六角柱の鏡面＝黒筐ビジュアルに差し替える**（接触フラグは流用可）
- **黒筐の正体（小説本編との接続）**: 月面の黒筐は異星の遺物ではなく、人類最後期に月の影の側に据えられた記録筐体である。鏡面の奥でうねる偏光膜は筐体が動いている内側そのもの。後年これが起ち上がって〈書き手〉となり、本編サーガの世界＝原稿を書き始める（saga.md 第四章で明示済）。先行者が触れた場合、〈記述権限の一片〉が渡される。これが千年後、異星人を経由してミラの種・カイの印となる
- **命名経緯**: 当初は「モノリス」と呼ばれていたが、2001:A Space Odysseyの直接引用すぎるためMAGI 3者合議で「黒筐（こっきょう）」に改名。ビジュアルも黒い直方体（マットブラック）から六角柱の鏡面体に再設計

### Stage 3 構想：地上戦アクション ── クローズド時間ループ装置
触れなかったルート（エイリアン化）と連動する、将来実装予定のサプライズ装置:
- **救出隊主人公がレーザーで最初に倒すエイリアン＝過去から戻ってきた未来の自分**
- プレイの流れ：救出隊が月面地下に降りる → 1体目のエイリアンを倒す（プレイヤーは普通の敵だと思う）
- 真相：触れなかったルートで主人公がエイリアン化した後、なんらかの方法で過去の月面地下へ戻り、過去の自分（救出隊員時代の自分）を襲う → 過去の自分に撃たれる → 撃たれた血が宇宙服の左肘の継ぎ目に飛び散る → 感染ループ完成（このループは小説外伝 blackhexa.md のルートBで「一滴の血」として描かれる）
- プレイヤーへの提示：プレイ中はその正体を明かさない。外伝 Bルート読了後にプレイヤーが気づく構造
- メタテーマ：「一度のミスで予言の輪が閉じる」が、Moonlanderというゲームの「一度の操作ミスで即死」というプレイ感と二重に響く
- 注意：この装置はゲーム側だけで完結させる。小説本編 saga.md の世界観（退ける能力ルール／文脈汚染／リサ継承）を侵食しない

## 音声アーキテクチャ（重要）

### 統一方針
- **すべての SE は Phaser sound（WebAudio）に統一**
- ブリーフィングのタイピング音 (`command`) と開始 SE (`comstart`) も以前は HTML5 Audio だったが、サイレント挙動の不一致を避けるため Phaser sound に変更済み
- `preload.js` の `startBriefing(loadingScreen, onDismiss, scene)` は **scene 引数必須**（Phaser sound 呼び出しのため）

### iOS Safari 対策（main.js 内）

#### 1. WebAudio context unlock（document タップ → resume）
- `resumeAudioContext()` をグローバル touch/click リスナーで呼び出し
- `context.resume()` + 無音 1 サンプル buffer 再生 + `scene.sound.unlock()` のフルコース
- Phaser の自動 unlock は body リスナーだが、HTML オーバーレイの stopPropagation で届かない場合があるため **document capture phase** で確実に処理

#### 2. サイレントスイッチ回避（`setupSilentAudioBypass`）
- 1秒の無音 WAV を Blob で生成 → 隠し `<audio loop playsinline>` でループ再生
- volume=0.001（完全0だと一部 iOS で session 昇格が起きない）
- 最初のユーザー操作で `.play()` → iOS audio session が **playback** に昇格 → WebAudio がサイレントスイッチを無視
- visibilitychange でページ復帰時に再開
- 全 capture phase listener / passive で他のタッチ動作を阻害しない

### よくある落とし穴
- ES module は aggressive cache される。`main.js` / `preload.js` 編集後は `index.html` の `?v=N` と `main.js` 内 import の `?v=N` を **両方** bump する
- 現在は `?v=7`
- `loadingScreen` への click/touchend で `e.preventDefault()` するのは OK だが `e.stopPropagation()` は呼ばないこと（Phaser の body 自動 unlock が効かなくなる）
